# 实时协作协议契约

- 服务端按文档流分配单调递增的 `streamSequence`，客户端检测重复和缺口。
- 命令拒绝是请求响应，presence 使用每客户端短期版本；二者不占用可恢复文档序列。
- 命令包含全局唯一 `commandId` 与 `baseRevision`；重试沿用同一 ID。
- 服务端是权限、校验和持久化结果的权威来源，确认消息返回规范化文档。
- 快照必须携带其覆盖的序列位置；应用快照后才能丢弃旧操作。
- presence 是可过期提示，不参与持久文档的一致性与权限判断。
- 重连先鉴权，再从最后确认序列恢复；历史已裁剪时回退到新快照。
- 客户端对待发送字节、乱序事件和离线命令设置容量上限。
- 所有协议消息在运行时校验，未知版本或未知消息不得进入状态层。
