# 如何使用这个学习站

本站采用“理解问题 → 学习概念 → 运行代码 → 修改示例 → 放进项目 → 间隔复习”的方式推进。

## 每一节怎么学

1. 先阅读学习目标和前置知识，确认本节边界。
2. 不复制代码，先预测输出，再亲手运行。
3. 用自己的话解释代码执行过程。
4. 修改示例中的输入、类型或分支，观察编译结果和运行结果。
5. 把错误和疑问记录到学习笔记。
6. 24 小时后进行一次不看文档的快速复习。

## 什么叫“学会了”

能看懂不等于能使用。至少满足下面四点，才算完成一个知识点：

- 能用自己的语言解释它解决的问题。
- 能写出一个最小示例并预测结果。
- 能识别一个常见错误并说明原因。
- 能在真实项目中判断什么时候应该使用它。

## 建议节奏

一次学习 30～90 分钟即可：

- 10～20 分钟：阅读概念。
- 20～40 分钟：运行和修改示例。
- 15～25 分钟：修改示例并验证自己的理解。
- 5 分钟：总结和记录下一步。
