# 人工智能、机器学习、深度学习与生成式 AI 的关系

> 本文讨论稳定的基础概念，不依赖某个模型、框架或产品版本。文中的集合关系是学习用的近似地图，不是所有学术与法规语境下唯一的分类标准。

## 1. 学习目标

完成本节后，你应该能够：

- 用自己的话解释人工智能、机器学习、深度学习和生成式 AI。
- 说明为什么机器学习不等于人工智能，深度学习也不等于机器学习的全部。
- 区分“由人直接写规则”和“从数据中学习参数”。
- 区分判别任务与生成任务。
- 用训练、模型和推理描述一个机器学习系统的基本流程。
- 看懂监督学习和自回归语言模型最基本的数学表达。

## 2. 先看全景：四个词不在同一层

最常用的入门地图是：

```text
人工智能（AI）
├── 基于人工知识的方法：规则、搜索、规划……
└── 机器学习（ML）
    ├── 传统机器学习：线性模型、决策树、支持向量机……
    └── 深度学习（DL）
        ├── 判别式模型：分类、检测、排序……
        └── 现代生成式 AI：文本、图像、音频、视频生成……
```

这张图表达三个关键事实：

1. **AI 是目标与系统层面的总称。** 系统根据输入产生预测、内容、建议或决策，以完成某个目标。
2. **机器学习是实现 AI 的一组方法。** 它从数据或交互经验中调整模型，而不是由开发者穷举每一条判断规则。
3. **深度学习是机器学习的一类方法。** 它使用多层参数化模型学习逐层抽象的表示。

生成式 AI 的分类角度不同：它强调系统**产生什么输出**，而机器学习和深度学习更强调系统**如何学到能力**。现代生成式 AI 大多由深度学习实现，但“生成式”不是“深度学习”的同义词，深度学习也大量用于分类、检测和预测。

因此，“AI ⊃ ML ⊃ DL”可以帮助入门，但不要把所有概念都强行画成严格的同心圆。

## 3. 什么是人工智能

“像人一样思考”很直观，却不是一个容易检验的工程定义。更实用的观察方式是：

> 一个机器系统接收输入，围绕显式或隐式目标进行推断，再产生会影响物理或虚拟环境的预测、内容、建议或决策。

这与 OECD 2023 年更新的 AI 系统定义保持一致。它把重点放在**输入、目标、推断、输出和环境**，并没有要求系统必须拥有意识，也没有规定系统必须使用神经网络。

例如：

| 系统 | 输入 | 输出 | 可能采用的方法 |
| --- | --- | --- | --- |
| 国际象棋程序 | 当前棋盘 | 下一步棋 | 搜索、规则、机器学习 |
| 垃圾邮件过滤器 | 邮件内容 | 垃圾/正常 | 规则或分类模型 |
| 商品推荐系统 | 浏览与购买记录 | 推荐列表 | 协同过滤、深度学习 |
| 文本生成系统 | 提示词与上下文 | 后续文本 | 生成式语言模型 |

AI 是不断变化的研究与工程领域，边界会随语境变化。与其争论某个简单程序“算不算 AI”，不如明确它的任务、输入输出、知识来源、自治程度和局限。

## 4. 规则系统与机器学习

假设我们根据学习时长预测一名学习者能否完成课程。

规则系统由人直接指定判断方式：

```python
def predict_with_rule(hours: float) -> bool:
    return hours >= 6
```

机器学习则先提供带答案的样本，让算法选择表现较好的模型参数。仍然使用“时长阈值”这个极简模型时，可以写成：

\[
f_t(x) = \begin{cases}
1, & x \ge t \\
0, & x < t
\end{cases}
\]

开发者规定了模型形式 \(f_t\)，训练过程根据数据选择阈值 \(t\)。这说明机器学习并不是“完全没有人为规则”：人仍然决定任务、数据、特征、模型族、损失函数和评估方式；学习算法只在这些边界内寻找参数。

更一般地，监督学习拥有训练集：

\[
D = \{(x_i, y_i)\}_{i=1}^{n}
\]

模型 \(f_\theta\) 的参数为 \(\theta\)。训练常被写成最小化经验风险：

\[
\theta^* = \arg\min_\theta \frac{1}{n}\sum_{i=1}^{n}
\mathcal{L}(f_\theta(x_i), y_i)
\]

其中 \(\mathcal{L}\) 是损失函数，用来衡量预测与目标之间的差异。训练得到 \(\theta^*\) 后，把新输入交给模型得到输出，称为**推理**（inference）。

```text
训练：训练数据 + 学习算法 → 已训练模型
推理：新输入 + 已训练模型 → 预测或生成结果
```

配套文件 `examples/ai/foundations/learning_vs_rules.py` 展示了规则阈值和数据学习阈值的差别。请在仓库根目录运行：

```bash
python3 examples/ai/foundations/learning_vs_rules.py
```

## 5. 什么是深度学习

深度学习使用包含多个处理层的模型，从数据中学习多层表示。一个前馈神经网络可以抽象为：

\[
h^{(0)} = x
\]

\[
h^{(l)} = \sigma(W^{(l)}h^{(l-1)} + b^{(l)}),
\quad l = 1, 2, \ldots, L
\]

\[
\hat{y} = g(h^{(L)})
\]

这里：

- \(x\) 是输入；
- \(W^{(l)}\) 和 \(b^{(l)}\) 是第 \(l\) 层需要学习的参数；
- \(\sigma\) 是非线性激活函数；
- \(h^{(l)}\) 是逐层形成的表示；
- \(\hat{y}\) 是模型输出。

“深”主要指多个表示层，而不是“像人一样深度思考”。图像模型可能从局部边缘逐步形成更复杂的视觉表示；语言模型则把 Token 逐步转换为结合上下文的信息表示。

深度学习的优势通常来自表示学习：模型可以从原始或较少加工的数据中学习适合任务的特征。但这并不意味着深度学习永远优于传统方法。小规模表格数据、需要强可解释性或计算资源有限时，线性模型、树模型等往往仍是更合适的基线。

## 6. 什么是生成式 AI

生成式 AI 的目标是学习输入数据的结构与统计规律，再产生新的合成内容，例如文本、图像、音频、视频或代码。

### 判别与生成关注的问题不同

- **判别任务**：这封邮件属于哪个类别？这张图里有没有猫？
- **生成任务**：生成一封邮件摘要；根据描述生成一张猫的图片。

对于类别 \(y\) 和输入 \(x\)，判别模型常直接学习 \(p(y\mid x)\) 或一个决策边界。生成模型则对数据分布建模，例如 \(p(x)\)、联合分布 \(p(x,y)\)，或条件生成分布 \(p(y\mid x)\)。因此，符号 \(p(y\mid x)\) 本身不能决定模型是不是“生成式”；关键还要看 \(y\) 表示类别标签还是要生成的复杂内容，以及模型如何建模和采样。

### 语言模型为什么能生成文本

自回归语言模型把一段 Token 序列的概率分解为：

\[
p(x_1, x_2, \ldots, x_T)
= \prod_{t=1}^{T} p(x_t \mid x_1, \ldots, x_{t-1})
\]

训练时，模型反复学习“给定前文，下一个 Token 的概率是多少”。推理时，模型从这个概率分布中选择或采样下一个 Token，再把它加入上下文并继续预测，于是形成连续文本。

Transformer 是现代语言模型常用的神经网络架构，但需要分清：

- Transformer 是一种**模型架构**；
- 语言建模是一个**学习目标/任务**；
- 大语言模型是经过大规模训练得到的**模型类别**；
- 聊天助手是组合模型、提示、检索、工具、安全策略和界面的**完整系统**。

模型输出在统计上合理，不等于内容真实。流畅文本、事实正确性和可靠行动是不同的评估维度。

## 7. 用五个问题识别一个 AI 系统

面对新术语或新产品时，先问：

1. **任务是什么？** 分类、预测、排序、生成，还是控制？
2. **输入和输出是什么？** 文本、图像、数值、动作，还是多种模态？
3. **知识从哪里来？** 人写规则、训练数据、环境反馈，还是它们的组合？
4. **训练阶段优化什么？** 使用什么目标函数、数据和反馈信号？
5. **推理阶段如何评估？** 准确率、误差、生成质量、延迟、安全性，还是业务效果？

这五个问题比只记住产品名更稳定，也能帮助你判断后续需要学习哪一类知识。

## 8. 常见误解

### “只要是 AI，就一定会学习”

不一定。基于规则、搜索或规划的系统也可能属于 AI。机器学习只是其中一类方法。

### “机器学习就是神经网络”

不是。线性回归、逻辑回归、决策树、支持向量机和聚类等都属于机器学习，但通常不称为深度学习。

### “生成式 AI 就是大语言模型”

不是。大语言模型主要生成和处理语言 Token；生成式 AI 还包括图像、音频、视频等模型，也包括 GAN、变分自编码器和扩散模型等不同技术路线。

### “模型越大，就越智能”

参数规模只是影响能力、成本和部署方式的因素之一。数据质量、训练目标、架构、推理策略、工具、评估和使用场景同样重要。

### “训练完成后，模型会自动持续学习”

通常不会。大多数部署模型在一次推理中参数保持不变。上下文中的新信息会影响本次输出，却不等于模型权重已经更新。是否进行在线学习或后续训练，取决于系统设计。

## 9. 稳定原理、版本行为与工程实践

后续学习中要主动区分三类信息：

| 类型 | 本课示例 | 应该如何对待 |
| --- | --- | --- |
| 稳定原理 | 经验风险最小化、训练与推理的区别、自回归分解 | 建立长期可迁移的理解 |
| 版本行为 | 某个框架 API、模型支持的上下文长度 | 查阅对应版本的官方文档 |
| 工程实践 | 数据划分方式、监控指标、部署方案 | 结合数据、成本和风险验证 |

本课重点是稳定原理。等进入 NumPy、scikit-learn、PyTorch 和 Transformers 实作时，会明确标注适用版本。

## 10. 本课小结

- 人工智能是最宽的系统与研究领域，既包含学习方法，也包含知识驱动的方法。
- 机器学习从数据或经验中选择模型参数，是实现 AI 的重要方法。
- 深度学习通过多层参数化模型学习表示，是机器学习的一部分。
- 生成式 AI 按输出目标来描述系统；现代生成式 AI 通常建立在深度学习之上，但两者不能画等号。
- 训练负责学习参数，推理负责使用已训练模型产生预测或内容。
- 理解一个 AI 系统时，应同时观察任务、数据、模型、目标、推理和评估。
