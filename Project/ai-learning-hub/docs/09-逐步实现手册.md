# 逐步实现手册

> 本手册是后续实现时的唯一执行顺序。每完成一步，先运行验证，再阅读下一步；不同时引入多个新概念。

## 简化原则

第一版保留必需的 Vue 3、Java、JWT、MySQL 和管理/学习闭环，但主动减少复杂度：一个前端应用、一个 Spring Boot 应用、一个 MySQL 数据库；Redis、消息队列、微服务、容器生产部署和 AI 功能都不作为起步条件。Redis 仅在确实需要多实例刷新令牌共享或限流时再加入。

## 执行步骤

| 步骤 | 学习目标 | 实现产物 | 验证方式 | 停止点 |
| --- | --- | --- | --- | --- |
| 0 | 理解边界与需求 | 阅读本目录设计文档，确认开放决策 | 需求和角色能用自己的话复述 | 用户批准进入工程初始化。 |
| 1 | 学会前后端最小启动 | `web/`、`server/` 的 Hello/health 页面与 README | 浏览器打开前端；`/actuator/health` 返回 UP | 不加入数据库和登录。 |
| 2 | 学会 API 契约 | `contracts/openapi.yaml` 的健康、认证和内容读取草案 | OpenAPI 校验；前端请求一个 mock/真实健康接口 | 确认前后端同域或跨域方案。 |
| 3 | 学会数据库迁移 | MySQL 本地配置、Flyway 基线、用户/角色/刷新令牌表 | 空数据库可完成迁移；重复迁移安全 | 不创建所有业务表。 |
| 4 | 学会认证与授权 | 注册、登录、JWT、当前用户、角色守卫 | 学习者可登录；无令牌 401；非管理员 403 | 已确定 JJWT、短 access token + HttpOnly refresh Cookie；不做邮箱验证和密码找回。 |
| 5 | 学会内容建模 | 路线、课程、知识点的管理员 CRUD 与公开浏览 | 草稿不可访问；发布后可见；排序正确 | 准备至少一套种子内容。 |
| 6 | 学会状态与个人数据 | 日计划、任务、完成、学习时长 | 同一用户当天仅一份计划；完成可幂等；时长合法 | 确认连续学习日的时区口径。 |
| 7 | 学会个人知识沉淀 | 笔记、标签、搜索/筛选 | 用户不能读取他人的笔记；标签去重 | 决定笔记能否独立存在。 |
| 8 | 学会数据展示 | 进度与仪表盘四项指标 | 指标能从任务/时长记录复算；无数据显示引导 | 确认掌握度文案和计算规则。 |
| 9 | 学会交付质量 | 单元/集成/E2E 测试、日志、备份恢复、发布候选 | P0 场景全通过；恢复演练成功 | 评审是否进入试用。 |

## 每一步的固定流程

1. 先读对应设计文档和前置知识；
2. 只实现该步骤列出的最小闭环；
3. 写或更新测试，再运行验证命令；
4. 在实现记录中写出“手把手搭建”部分：文件路径、创建顺序、关键代码、每段代码的作用与验证命令；
5. 把结果发给用户确认，再开始涉及新领域模型或安全策略的下一步。

## 实现记录的教学模板

从步骤 5 起，每份实现记录除了说明结果，还必须可以让你在空白分支上跟做。推荐按下面顺序阅读和操作：

1. **先理解**：用一句话说明本步要解决的业务问题与新概念；
2. **文件地图**：列出会新增或修改的文件，先分清数据库、后端、前端各自职责；
3. **按顺序编码**：一次只创建一个小单元，给出关键代码片段并解释输入、输出与原因；
4. **立即验证**：每一小段都给出最短验证命令、预期 HTTP 状态或界面现象；
5. **复盘边界**：说明为何不实现相近但更复杂的功能，以及下一步从哪里继续。

代码片段不是要求机械复制。应先看注释理解它与前后代码的关系，再亲手输入或对照实现；涉及 `.env`、JWT、Cookie、密码和真实账号时，文档只使用占位符，绝不记录真实值。

## 必须先确认的选择

已确认采用**同域代理**：开发时浏览器访问 Vite，Vite 代理 `/api` 到后端；未来由 Nginx 承接同一职责。它能简化 JWT refresh Cookie、CORS 和 CSRF 的教学与实现。代理配置和验证步骤记录在 [M0 工程基线实现记录](10-M0-工程基线实现记录.md)。

## 范围、非目标、风险与验收

**范围**：把设计转换为小步学习和实现计划；每步都可独立验证。

**非目标**：不以“学习”为名跳过测试、安全或需求确认；不在没有批准时提前创建工程或引入 AI/Redis/微服务。

**风险**：步骤 4 的 JWT 与 Cookie 安全容易被低估，步骤 8 的指标规则容易反复变更。通过同域优先、最小 JWT 实现、先以原始记录计算指标来控制风险。

**验收**：实施时每个步骤必须有实际文件、可重复命令和结果记录；上一步验证失败时不进入下一步。
