## 这是什么

AI Learning Hub 是面向个人学习者与内容运营管理员的学习成长管理平台。第一版聚焦可追踪、可管理的学习闭环：内容组织、计划执行、记录沉淀和可解释的学习数据；AI 能力被明确放入第二阶段，不以占位功能进入第一版。

本目录是 `Project/ai-learning-hub/` 下的**独立产品与工程规划包**，不属于现有 VitePress 学习站的发布内容。现有站点继续以仓库根目录的 `docs/` 为源目录、`docs/.vitepress/config.ts` 为导航配置；本目录既不修改上述文件，也不参与现有站点的 `npm run build`。

## 已定边界与未来目录策略

```text
仓库根目录
├── docs/                              # 已有 VitePress 学习站：保留，不承载业务应用
├── examples/                          # 已有示例代码：保留
└── Project/ai-learning-hub/           # 本项目唯一工作边界
    ├── docs/                          # 当前规划与实现过程文档
    ├── web/                           # 未来：Vue 3 学习者端与管理端
    ├── server/                        # 未来：Spring Boot 模块化单体
    ├── infra/                         # 未来：本地环境与部署说明
    └── contracts/                     # 未来：OpenAPI 接口契约
```

未来若进入编码，所有应用源码均在 `Project/ai-learning-hub/` 下，而不是仓库根目录的 `docs/`。本项目采用简单的单项目目录，不引入 monorepo 工具链。

## 全局范围、非目标、风险与验收

**第一版范围**：账号与 RBAC、路线/课程/知识点、任务/日计划/完成记录/进度、笔记与标签、管理端内容管理、学习仪表盘。

**明确非目标**：AI 生成或问答、支付、社交/排行榜、证书、移动原生应用、多租户 SaaS、复杂审批流、与现有 VitePress 内容的自动同步。

**主要风险**：产品规则尚有待确认（任务如何关联内容、掌握度算法）；JWT 刷新与撤销策略会影响安全模型；学习时长与连续学习的口径会影响指标可信度；未来 AI/RAG 会引入数据授权、来源引用与成本治理。

**本规划包验收**：八份文档彼此链接一致；每份至少声明范围、非目标、风险和可验证的验收方式；第一版与第二阶段边界清楚；不改动 `docs/`、VitePress 配置或业务工程。
